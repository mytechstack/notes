  What each service owns                                                                                                                                                                                                                             
                                                            
  hydration-service:                                                                                                                                                                                                                                 
  - POST /hydrate only                                      
  - Validates JWT hydration token
  - Calls backend services       
  - Writes to Redis                                                                                                                                                                                                                                  
  - No session auth middleware
  - Can be in a DMZ or semi-public network zone                                                                                                                                                                                                      
                                                                                                                                                                                                                                                     
  context-reader-service:                                                                                                                                                                                                                            
  - GET /data/{userId}/{resource}                                                                                                                                                                                                                    
  - GET /context/{userId}                                                                                                                                                                                                                            
  - GET /health                                             
  - Validates session token on every request                                                                                                                                                                                                         
  - Reads from Redis only (no backend calls — cache miss = 404)
  - Sits behind your auth gateway                                                                                                                                                                                                                    
                                                                                                                                                                                                                                                     
  The cache-miss question                                                                                                                                                                                                                            
                                                                                                                                                                                                                                                     
  Your current handler_data.go falls back to a live backend call on cache miss. With the split, you need to decide:                                                                                                                                  
                                                                                                                                                                                                                                                     
  Option A — cache miss returns 404                                                                                                                                                                                                                  
  cache miss → 404                                          
  caller must handle it → trigger hydration → retry                                                                                                                                                                                                  
  Simpler, context-reader has zero backend dependency.                                                                                                                                                                                               
                                                                                                                                                                                                                                                     
  Option B — cache miss triggers re-hydration                                                                                                                                                                                                        
  cache miss → call hydration-service internally → wait → return data                                                                                                                                                                                
  Context-reader stays read-only externally but calls hydration-service internally.                                                                                                                                                                  
                                                                                                                                                                                                                                                     
  Option A is cleaner — the reader service has no backend knowledge at all. Redis is its only dependency. Cache miss tells the caller "not warm yet, hydration needed."
